## Decision log (UI/UX calls not pinned by the spec)

- **Transfer ownership uses severity='danger'** - The spec lists
  Transfer + Archive + Delete in the action matrix but only pins
  severity routing for Archive (warning) and Delete (danger). I picked
  danger for Transfer because ownership transfer is irreversible from
  the previous owner's side (the only path to undo is the new owner
  transferring back). Typed-name gate prevents fat-finger transfers.

- **Archive auto-closes the modal** - The panel calls `onClose?.()`
  after a successful Archive commit. This is consistent with Delete
  (which obviously closes the modal because the channel is gone) and
  matches the user expectation that the archived channel is no longer
  in the live list (so the modal's owned-channel projection should
  refresh). Required updating one polish-wire test that had assumed
  both Archive + Delete buttons remained reachable after Archive.

- **Inline rename + topic edit share the same role gate (canRename)** -
  Owners and admins can both rename a channel and edit its topic. The
  spec lists Rename as one action per row but the panel treats Topic
  Edit as a sibling micro-action under the same gate (it's a no-op
  rename from the user's perspective, just on a different field). The
  existing Wave 0 inline topic edit had no role gating; this gates
  topic edit behind canRename now.

- **Empty state copy ("You don't have admin rights in this channel.")** -
  Matches the Wave 0 stub's docstring exactly.

- **Hydrating placeholder copy ("Loading admin actions...")** -
  Renders when `currentChannelRole == null`. The spec said "skeleton
  placeholder"; I implemented a textual placeholder rather than a
  shimmer animation because the directory modal is gated on
  `hasOwnedChannels` and the prop is always 'owner' in practice until
  Wave B lands the role accessor, so this code path is reached only
  by tests today.

- **No `EMPTY_STATES.noTopicSet` lookup** - The panel hardcodes
  "No topic set" rather than importing the copy module. Reasoning:
  the panel is intentionally self-contained per the Wave 0 contract
  (no co-imports beyond lucide icons). Updated empty-states.spec to
  accept either string.
