## 1. Scope

Dropped the four-channel hardcoded seed list (`general`, `project-alpha`,
`lora-training`, `random`) from `MqttChatStore.channels` initial state and
replaced it with `$state([])`. Added a private `#bootstrapChannels()` helper
that calls `/api/conversations` (via the existing `apiGet` helper in
`lib/api.js`) after the MQTT broker handshake, maps each row into the
store-internal shape with the v0.4.0 `my`-prefix → unprefixed rename
(`myUnread → unread`, `myStarred → starred`, `myMuted → muted`), and
populates `channels` with the result. Added a reactive `serverUnreachable`
flag that flips to `true` on 404 / 500 / network failure (banner wiring in
App.svelte deferred to Step 2.6's wave per the architecture spec). Wired
the helper into `connect()`'s `'connect'` callback so bootstrap runs after
`#subscribeAll()` / `#publishPresence('online')` and before
`#drainPendingSends()` / `#fetchHistory()`. Added a test seam
(`_bootstrapChannelsForTest()`) and a new vitest spec covering 10 cases.

### Bootstrap helper + field rename

```diff
+  async #bootstrapChannels() {
+    let rows;
+    try {
+      rows = await apiGet('/api/conversations');
+    } catch (err) {
+      this.serverUnreachable = true;
+      this.channels = [];
+      this.#recordParseFailure();
+      console.warn('[claude-comms] /api/conversations bootstrap failed', { ... });
+      return;
+    }
+    let list = rows;
+    if (rows && !Array.isArray(rows) && Array.isArray(rows.conversations)) {
+      list = rows.conversations;
+    }
+    if (!Array.isArray(list)) {
+      this.channels = [];
+      this.serverUnreachable = false;
+      return;
+    }
+    this.channels = list.map((row) => this.#channelRowFromPayload(row));
+    this.serverUnreachable = false;
+  }
+
+  #channelRowFromPayload(row) {
+    const r = row && typeof row === 'object' ? row : {};
+    return {
+      id: ..., name: ..., topic: ..., member: ..., memberCount: ...,
+      lastActivity: ..., mode: ..., visibility: ...,
+      createdAt: ..., createdBy: ...,
+      // my-prefix → unprefixed rename (architecture spec §III.4 preamble)
+      unread: typeof r.myUnread === 'number' ? r.myUnread : 0,
+      starred: r.myStarred === true,
+      muted: r.myMuted === true,
+      archived: r.archived === true,
+      archived_at: r.archived_at ?? null,
+      archived_by: r.archived_by ?? null,
+    };
+  }
```

  uses `this.channels` as an array** (~17 call-sites use `.find()`,
  `.filter()`, `.some()`, `.map()` with index). Step 2.6 (architecture

  step's scope. Flagged so the orchestrator knows the array→map conversion
  remains a Step 2.6 deliverable.

  in `/api/conversations` (mcp_server.py lines 542–567 omit them). The
  brief listed them in the payload contract anyway. Store defaults
  defensively (`archived: false`, `archived_at: null`, `archived_by: null`).
  If the daemon later starts emitting these, the mapping picks them up
  with zero code changes. Test #9 (`defends against partial / missing
  fields`) verifies this.
