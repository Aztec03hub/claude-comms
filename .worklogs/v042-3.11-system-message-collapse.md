## WHAT shipped

A new `SystemMessageGroup.svelte` component collapses runs of 3 or more
consecutive system events (join / leave / archive / unarchive / delete /
topic_changed / renamed / member_joined / member_left) into a single muted
summary row with click-to-expand. Runs of 1 or 2 system events still render
inline, matching pre-3.11 behaviour. `MessageGroup.svelte` now detects when
its `messages` array is a system-only run and delegates to
`SystemMessageGroup`; non-system runs continue to render through
`MessageBubble` with zero behaviour change.

Visual design per the Step 3.11 spec:
- Muted color (`var(--text-faint)`), smaller font (11px summary / 12px event),
  no avatar.
- ChevronRight / ChevronDown affordance on the summary button.
- Expanded events are indented with a left border so the relationship is
  visible.
- Focus-visible outline using `--ember-400` matches the rest of the v0.4.x
  ember-themed UI.

The summary stitches the pre-humanized event bodies (Wave A Step 2.7's
daemon already renders join/leave/archive/etc. strings into `msg.body`
before publish) with commas and a trailing `(N events)` count — e.g.
`"Alice joined, Bob left, Carol archived #general (3 events)"`. No
re-implementation of the daemon humanizer, single source of truth.

## MessageGroup integration approach

`MessageGroup` derives a single boolean `isSystemGroup` predicate:

```
let isSystemGroup = $derived(
  messages.length > 0 &&
    messages.every((m) => m?.sender?.type === 'system')
);
```

The `every` (not `some`) keeps a malformed mixed run from silently
disappearing into the collapsed path — if any entry is non-system, the
whole group falls through to `MessageBubble`. Empty arrays are not
system groups (rendered as nothing via the each block, matching the
pre-3.11 behaviour).

This keeps the delegation boundary clean for the upcoming Wave E.2
`ChatView` rework: E.2 will replace the per-message `currentGroup = { type:
'system', messages: [msg] }` accumulator in `ChatView.svelte` with a true
run accumulator and route the resulting `group.messages` through
`MessageGroup` directly (or skip the intermediate group-type discriminator
entirely). The contract `MessageGroup` exposes ("a system-only array
renders as SystemMessageGroup") is stable across that future refactor.

## Expand / collapse interaction

- `expanded` is a component-local `$state(false)` boolean.
- The summary is a `<button type="button">` (not a `<div role="button">`)
  so it gets keyboard activation, focus management, and screen-reader
  semantics for free.
- `aria-expanded` and `aria-controls` are wired up correctly; the expanded
  events container has the matching `id="system-group-events"`.
- An explicit `onkeydown` handler still calls `toggleExpanded` on Enter /
  Space, defensively, since `fireEvent.keyDown` in tests doesn't always
  dispatch the synthetic click that a real browser would for `<button>`.
