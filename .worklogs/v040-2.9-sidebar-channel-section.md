## 1. Scope

Created a new reusable Svelte 5 component `SidebarChannelSection.svelte`
plus its companion vitest spec `tests/sidebar-channel-section.spec.js`.
The component renders a collapsible left-sidebar section composed of an
always-visible header (icon + label + count + chevron) and a body region
that either lists `SidebarChannelRow` instances (one per channel) or
shows a muted italic empty-state line when `channels.length === 0`.

The header is a `<button aria-expanded aria-controls={bodyId}>` that
toggles collapse state; the chevron rotates 0deg → 90deg via CSS, honoring
`prefers-reduced-motion`. Collapse state persists per-section through a
caller-supplied `storageKey` in `localStorage`, encoded as the same
`"1"`/`"0"` string format used by the MemberList M-FIX section toggles in
v0.3.3. The body is a `<div role="region" aria-label={label}>` whose
unique id is derived from the label so multiple sibling sections in the
Sidebar shell (Starred / Active / Available) never collide.

Section visibility is never gated on row count: the header always renders,
extending Phil's M-FIX invariant from MemberList into the new channel
sidebar surface.

A minimal placeholder `SidebarChannelRow.svelte` was also committed as a
build-gate workaround (see §9). Agent 2.8's deliverable must overwrite it
at integration time.

### b. localStorage persistence with `untrack` to silence the
`state_referenced_locally` lint

```svelte
let isExpanded = $state(
  untrack(() => readStoredBool(storageKey, defaultExpanded)),
);

$effect(() => {
  if (typeof localStorage === 'undefined' || !storageKey) return;
  try {
    localStorage.setItem(storageKey, isExpanded ? '1' : '0');
  } catch {
    // ignore — quota / disabled storage
  }
});
```

### c. The body region — empty-state line OR rows via the child

```svelte
{#if isExpanded}
  <div class="sidebar-channel-section-body"
       id={bodyId}
       role="region"
       aria-label={label}
       data-testid="sidebar-channel-section-body-{label}">
    {#if channels.length === 0}
      <div class="sidebar-channel-section-empty"
           data-testid="sidebar-channel-section-empty-{label}"
      >{emptyState}</div>
    {:else}
      {#each channels as channel (channel.id)}
        <SidebarChannelRow
          {channel}
          isActive={channel.id === activeChannelId}
          {onChannelClick}
          {onChannelContextMenu}
          {onStarToggle}
        />
      {/each}
    {/if}
  </div>
{/if}
```

### Minor — `$effect` autofixer suggestion is a false positive

The autofixer surfaces one suggestion against `SidebarChannelSection.svelte`:

  "You are calling a function inside an $effect. Please check if the
  function is reassigning a stateful variable [...] check if it could
  use `$derived` instead."

The function in question is `localStorage.setItem(...)`, which is a
non-reactive side effect (writing to browser storage), not a state
reassignment. `$effect` is the correct rune here; `$derived` cannot
perform IO. MemberList uses the same pattern in three places (lines
89–106 of MemberList.svelte). No action required.
