## 1. Scope

Created `web/src/lib/copy/emptyStates.js` as a single source of truth for
empty-state copy in the UI. Updated three consuming components
(`MemberList.svelte`, `ChatView.svelte`, `ChannelDirectoryModal.svelte`)
to import + render constants from the new module instead of hardcoding
strings. Added `web/tests/empty-states.spec.js` with 13 new tests
covering module-level invariants (every value is a string-or-function;
no string contains an em dash; `filterEmpty` template behavior; required
keys present) plus three integration suites verifying each consumer
component renders the centralized constant.

Per the brief, `SidebarChannelSection.svelte` is a passive consumer —
its empty-state copy is passed in as a prop by the parent
(`Sidebar.svelte`). Sidebar is OUT OF SCOPE for this step, but its
inline strings are already em-dash-free (verified by grep + the new
test's em-dash guard). A follow-up step in v0.4.x can swap
Sidebar.svelte's inline strings for `EMPTY_STATES.starred` /
`EMPTY_STATES.active` / `EMPTY_STATES.available` — see §9.

  end with periods, providing the sentence cadence. This matches the
  pre-existing pattern in `ChatView.svelte`.
