## 1. Scope

Three UX gaps closed in a single landing:

1. **G-13 ŌĆö clickable toasts:** `NotificationToast.svelte` is now a card-shaped `<button>`. Clicking anywhere on the card fires an `onActivate({ channel, messageId })` callback which the parent (App.svelte) wires to `store.switchChannel(channel)` plus a defensive `store.goToMessage?.(messageId)`. The close-X is a nested `role="button"` span (HTML forbids nested native buttons) with `stopPropagation` so dismissing never routes.
2. **G-14 ŌĆö cap and coalesce:** App.svelte's toast handler now caps visible toasts at `TOAST_CAP = 3`. A 4th toast from the **same channel** coalesces in place into "`<sender> and N others sent messages`" (preserving channel-routing). Once a coalesced toast reaches 5+ events it promotes to a compact pill "`+N new in #channel`". Cross-channel 4ths displace the oldest (FIFO). The 5-second self-destruct resets on every coalesce.
3. **G-43 follow-up ŌĆö "Set your name" banner:** When `store.nameUnset === true`, App.svelte renders a one-line banner at the top of the main pane (above `<header.chat-header>`) with an ember-400 left border and an `ŌåÆ Open settings` action that flips `showSettingsPanel`. The banner is dismissible; dismissal persists under `localStorage` key `claude-comms.nameBanner.dismissed` and re-suppresses across reloads. The banner auto-disappears once `nameUnset` flips false (regardless of dismissal state).

### 4.2 G-14 ŌĆö coalesce algorithm in App.svelte

```diff
- function addToast(toast) {
-   toasts = [...toasts, toast];
-   setTimeout(() => {
-     toasts = toasts.filter(t => t.id !== toast.id);
-   }, 5000);

+ const TOAST_CAP = 3;
+ const COALESCE_TO_PILL_AT = 5;
+ const toastTimers = new Map();
+
+ function scheduleToastExpiry(id) {
+   const existing = toastTimers.get(id);
+   if (existing) clearTimeout(existing);
+   const handle = setTimeout(() => {
+     toastTimers.delete(id);
+     toasts = toasts.filter(t => t.id !== id);
+   }, 5000);
+   toastTimers.set(id, handle);
+ }
+
+ function clearToastTimer(id) {
+   const handle = toastTimers.get(id);
+   if (handle) { clearTimeout(handle); toastTimers.delete(id); }
+ }
+
+ function addToast(toast) {
+   const sameChannelIdx = toasts.findIndex(t => t.channel === toast.channel);
+   if (sameChannelIdx >= 0) {
+     const existing = toasts[sameChannelIdx];
+     const coalescedCount = (existing.coalescedCount ?? 1) + 1;
+     if (coalescedCount >= COALESCE_TO_PILL_AT) {
+       const updated = {
+         ...existing, pill: true, coalescedCount,
+         text: `+${coalescedCount} new in #${toast.channel}`,
+         messageId: toast.messageId ?? existing.messageId,
+       };
+       toasts = toasts.map((t, i) => (i === sameChannelIdx ? updated : t));
+     } else {
+       const others = coalescedCount - 1;
+       const updated = {
+         ...existing, pill: false, coalescedCount, sender: toast.sender,
+         text: `${toast.sender?.name ?? 'someone'} and ${others} other${others === 1 ? '' : 's'} sent messages`,
+         messageId: toast.messageId ?? existing.messageId,
+       };
+       toasts = toasts.map((t, i) => (i === sameChannelIdx ? updated : t));
+     }
+     scheduleToastExpiry(existing.id);
+     return;
+   }
+   let next = toasts;
+   if (next.length >= TOAST_CAP) {
+     const evicted = next[0];
+     clearToastTimer(evicted.id);
+     next = next.slice(1);
+   }
+   const fresh = { ...toast, coalescedCount: 1, pill: false };
+   toasts = [...next, fresh];
+   scheduleToastExpiry(fresh.id);
+ }
+
+ function dismissToast(id) {
+   clearToastTimer(id);
+   toasts = toasts.filter(t => t.id !== id);
+ }
+
+ function handleToastActivate(detail) {
+   if (!detail?.channel) return;
+   store.switchChannel(detail.channel);
+   if (detail.messageId && typeof store.goToMessage === 'function') {
+     store.goToMessage(detail.messageId);
+   }
+ }
```

## 8. Out of scope / deferred

- **`store.goToMessage(messageId)`:** the architecture spec mentions optional message-scroll routing on toast click, but the store doesn't expose `goToMessage` yet. `handleToastActivate` calls it defensively (`typeof store.goToMessage === 'function'`) so a later step that adds the helper light up the feature without further toast changes.
- **MqttChatStore reactivity in tests:** the mock store uses plain properties instead of `$state` (which can't be invoked outside Svelte component compilation). This means tests cannot exercise the App's notification `$effect` end-to-end by mutating `store.messages`. The 4 G-14 tests therefore verify the coalesce **contract** by rendering `NotificationToast` directly with the expected props (pill, coalesced text strings, channel routing). The full algorithm is exercised at integration time once the live MQTT broker plays real messages.
