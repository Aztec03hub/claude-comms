## 1. Scope

Two new MCP tools land the "Archive + kick" half of Phil's Q1 decision for the v0.4.0 conversation-lifecycle pass:

* `comms_conversation_archive(conversation, confirm=False)` — creator-only two-phase commit. `confirm=False` returns `{"error": "confirm_required", "message_count", "member_count"}` for the web client's blast-radius modal. `confirm=True` flips `archived=True` on `meta.json` (preserving history), ejects all members from the registry, retained-clears every member's per-conversation presence topic, and publishes a non-retained `{"type": "archived", ...}` event on `claude-comms/system/conversations`.
* `comms_conversation_unarchive(conversation)` — creator-only state reversal. Clears `archived` / `archived_at` / `archived_by` on `meta.json` and publishes a non-retained `{"type": "unarchived", ...}` event. Does NOT auto-re-join previously evicted members (Design Spec §4.4 — members re-join via their own `comms_join`).

Plus the supporting plumbing:

* `ConversationMeta` gained `archived_at: str | None` + `archived_by: str | None` fields and two thin transition methods (`mark_archived(archived_by)` / `mark_unarchived()`).
* `tool_comms_send` gained an `archived_guard` — when the caller wires `conv_data_dir`, sends into an archived conversation are rejected at the MCP layer with `{"error": "conversation_archived", ...}`. Server wiring always passes `conv_data_dir`; legacy / unit-test call sites that don't are unchanged.
* `tool_comms_conversations(all=True)` now surfaces `archived` / `archived_at` / `archived_by` on each `all_conversations[]` entry so Step 2.1's directory view can route archived conversations into the Archived sub-tab without a second round-trip.
* `mcp_server.py` gained a `_publish_archive_event(event_type, conversation_id, archived_by, evicted_keys)` helper that fans out the system event + per-member retained presence clears, swallowing per-publish exceptions so a single broker hiccup doesn't abort the fan-out.

Authorization model: creator-only for v0.4.0 (compared on `meta.created_by == participant.name`). The plan calls out a v0.4.1 admin-pass extension; not touched here.

## 5. Behaviour matrix

| Tool call | Result |
|---|---|
| `comms_conversation_archive(conv, confirm=False)` | `{"error": "confirm_required", "message_count": N, "member_count": M, ...}` — no state change |
| `comms_conversation_archive(conv, confirm=True)` as creator | `{"archived": True, "conversation_id": ..., "archived_by": ..., "archived_at": ..., "evicted_keys": [...]}` — meta flipped, members ejected, system event published, presence retained-cleared |
| `comms_conversation_archive(conv, confirm=True)` as non-creator | `{"error": "not_authorized", ...}` — no state change |
| `comms_conversation_archive("general", confirm=True)` | `{"error": "invalid_target", ...}` — reserved-name guard |
| `comms_conversation_archive(<unknown>, confirm=True)` | `{"error": True, "message": "...not found"}` |
| `comms_conversation_archive(<already-archived>, confirm=True)` | `{"archived": True, "status": "already_archived"}` — idempotent, no fan-out |
| `comms_send` into archived conv | `{"error": "conversation_archived", "conversation_id": ...}` |
| `comms_conversation_unarchive(conv)` as creator | `{"archived": False, "conversation_id": ...}` — meta cleared, system event published, members NOT re-joined |
| `comms_conversation_unarchive(conv)` as non-creator | `{"error": "not_authorized", ...}` |

## 10. Known risks / sharp edges

* **Idempotent archive returns `already_archived` without `evicted_keys`.** The server wrapper only publishes the system event + retained-clears presence when `result.get("evicted_keys") is not None`. This means a second archive call on an already-archived conversation will NOT re-publish the event nor re-clear presence — by design, but worth flagging in case a client wants to "force" a presence re-clear (it can't via this API).
* **Reserved-name guard fires even on `general` if a meta exists.** v0.4.0 doesn't allow archiving `general` regardless of creator. Test `test_archive_reserved_name_rejected` pins this.
* **`tool_comms_send` archived-guard requires `conv_data_dir` wired.** Legacy unit-test call sites that don't pass it keep the historical non-checking behaviour. Production wiring in `mcp_server.py` always passes it; reviewed every call site in this commit.
* **`mark_archived` is not idempotent on `archived_at`** — a second call refreshes the timestamp. The tool layer guards against re-archive via the `meta.archived` early-return, so the only way to hit a refreshed timestamp is to call `mark_archived` directly on a model. Acceptable: the model method is internal-ish, the tool is the public surface.
