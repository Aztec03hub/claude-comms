### 4b. New `$derived` block — three-section sidebar projections

```js
starredChannels = $derived(
  Object.values(this.channelsById)
    .filter((c) => c.member && c.starred)
    .sort((a, b) => (a.name || a.id || '').localeCompare(b.name || b.id || '')),
);

activeChannels = $derived(
  Object.values(this.channelsById)
    .filter((c) => c.member && !c.starred && !c.archived)
    .sort((a, b) => (a.name || a.id || '').localeCompare(b.name || b.id || '')),
);

availableChannels = $derived(
  Object.values(this.channelsById)
    .filter((c) => !c.member && c.visibility === 'listed' && !c.archived)
    .sort((a, b) => (a.name || a.id || '').localeCompare(b.name || b.id || '')),
);

archivedChannels = $derived(
  Object.values(this.channelsById)
    .filter((c) => c.archived === true)
    .sort((a, b) => (a.name || a.id || '').localeCompare(b.name || b.id || '')),
);
```

All four sections alpha-sort by `name` with `id` fallback. The
comparator is `localeCompare` so accent-bearing names alphabetize the
way users expect them to. Matches the SORT-LOCK invariant from the
architecture spec §III.4 preamble: alphabetical at every scale, every
section.

### 4c. 15-second undo wrapper

The wrapper takes three callbacks (optimistic / rollback / commitMcp).
The optimistic fn runs synchronously so the row visually leaves its
section the moment the user clicks; the MCP call is deferred by
`UNDO_WINDOW_MS`; if the user clicks Undo before the window elapses,
the rollback fn runs and the MCP call is never made.

```js
#scheduleUndoable(optimisticFn, rollbackFn, commitMcpFn) {
  try {
    optimisticFn();
  } catch (err) { /* surface as rejected envelope */ }

  let cancelled = false, committed = false, timer = null, resolveDone;
  const done = new Promise((resolve) => { resolveDone = resolve; });

  timer = setTimeout(async () => {
    timer = null;
    if (cancelled) return;
    committed = true;
    try {
      const result = await commitMcpFn();
      resolveDone({ ...result, cancelled: false });
    } catch (err) { /* ... */ }
  }, UNDO_WINDOW_MS);

  const cancel = () => {
    if (committed) return { tooLate: true };
    if (cancelled) return { tooLate: false };
    cancelled = true;
    if (timer) { clearTimeout(timer); timer = null; }
    try { rollbackFn(); } catch (err) { /* log + carry on */ }
    resolveDone({ success: true, cancelled: true });
    return { tooLate: false };
  };

  return { done, cancel };
}
```

`cancel()` returns `{ tooLate }` so the Step 2.12 context-menu invoker
can show a different toast variant ("Too late to undo") when the user
hits Undo after the MCP call already committed.

## 5. Lifecycle methods — argument shapes (decision table)

| Method | Async? | MCP tool | Confirms | Returns |
|---|---|---|---|---|
| `joinChannel(id)` | yes | `comms_join` | n/a | `Promise<{success, error?}>` |
| `leaveChannel(id)` | undo | `comms_leave` | n/a | `{done, cancel}` |
| `archiveChannel(id)` | undo | `comms_conversation_archive` | `confirm=true` | `{done, cancel}` |
| `deleteChannel(id)` | yes | `comms_conversation_delete` | `confirm=true` | `Promise<{success, error?}>` |
| `closeChannel(id)` | undo | delegates to `archiveChannel` | n/a | `{done, cancel}` |
| `setTopic(id, t)` | yes | `comms_conversation_update` | n/a | `Promise<{success, error?}>` |
| `setMute(id, level)` | no | (none — Q4 local lock) | n/a | `{success, error?}` |
| `setStar(id, b)` | no | (none — Q4 local lock) | n/a | `{success, error?}` |

All optimistic methods snapshot the pre-change row state and revert on
MCP error. `deleteChannel` is destructive (no undo per spec) but still
restores the row from a snapshot if the MCP call fails. `leaveChannel`
auto-unstars on the optimistic step (Design Spec §2.6: star is a
member-only decoration) and re-stars on rollback if the prior state was
starred.
