## 1. Scope

Added a `/`-prefix slash command registry + parser as a `.svelte.js` rune
module, registered the 12 v0.4.0 commands from the Heritage Survey table
(`/join`, `/leave`, `/list`, `/topic`, `/close`, `/star`, `/mute`, `/me`,
`/clear`, `/help`, `/who`, `/nick`), and wired
`MessageInput.svelte` to intercept `/`-prefixed input and dispatch
through the registry before the existing send / `/dm` / `/reply`
pipeline. Added 49 unit tests (target: 14) covering the pure parser, the
generic `SlashCommandRegistry` class, each of the 12 default commands,
and a "exactly 12 commands" coverage assertion.

The `/dm` and `/reply` inline parsers were preserved unchanged; they
synthesise a regular outbound message with recipients / replyTo metadata
and have their own dedicated parsers. The new interception path carves
them out via a `SLASH_INLINE_PARSERS` set so they never reach the
registry.

App-level side effects (`openDirectory`, `updateName`) are surfaced as
`slashCommand` CustomEvents on the composer root, mirroring the existing
`requestToast` decoupling pattern from `convertToArtifact`. The full
App.svelte listener wiring is intentionally deferred — the slash
commands work as a unit today via the toast surface; the trigger-based
modal opens are a small follow-up tracked in §7.

## 4. Code changes (samples)

### A — Parser entry point (new module)

```javascript
export function parseSlashCommand(input) {
  if (typeof input !== 'string') return null;
  if (!input.startsWith('/')) return null;
  const trimmed = input.trim();
  if (trimmed === '/') return null;
  const firstSpace = trimmed.indexOf(' ');
  const name = firstSpace === -1
    ? trimmed.slice(1)
    : trimmed.slice(1, firstSpace);
  const args = firstSpace === -1
    ? ''
    : trimmed.slice(firstSpace + 1);
  return { name: name.toLowerCase(), args };
}
```

### B — Registry class (new module)

```javascript
export class SlashCommandRegistry {
  commands = $state({});

  register(name, handler, { usage = '', description = '' } = {}) {
    this.commands[name] = { handler, usage, description };
  }

  async execute(input, context = {}) {
    const parsed = parseSlashCommand(input);
    if (!parsed) return { handled: false };
    const cmd = this.commands[parsed.name];
    if (!cmd) {
      return {
        handled: true,
        error: `Unknown command: /${parsed.name}. Try /help.`,
      };
    }
    const result = await cmd.handler(parsed.args, context);
    return { handled: true, ...(result ?? {}) };
  }
  // ... list() / get() ...
}
```

## 7. Findings surfaced during work

- (S) **App.svelte listener for the `slashCommand` CustomEvent is not
  wired in this step.** `/list` and `/nick` emit the event today but
  App.svelte does not consume it yet, so those two commands surface only
  their toast (the action is a no-op). Wiring is a small follow-up: add
  a `slashCommand` listener on the MessageInput container in App.svelte
  that opens `ChannelDirectoryModal` for `openDirectory` and calls
  `api.updateName(store.userProfile.key, value)` for `updateName`. ≈ 20
  lines. Suggest folding into Step 2.13 polish or a v0.4.0 follow-up
  micro-step. Tests already pin the trigger surface so the listener
  wiring is independently verifiable.

- (S) **`/me` styling deferred to v0.4.x render-side work.** The slash
  command sends a message with `kind: 'action'` via `store.sendMessage`,
  but `MessageBubble` doesn't yet differentiate on `kind`. The italic +
  ember accent rendering will land alongside the v0.4.1 message render
  refresh. The data path is complete today.

- (S) **`/clear` is a confirm-prompted stub** (per spec line 1974: "user-
  confirm"). The real `store.clearLocalBuffer(channelId)` method does
  not exist yet; spec'd it as v0.4.x follow-up. The slash command
  surfaces the deferral copy explicitly so the user is not misled.

- (S) **Inline help palette (typing `/` alone) is not implemented.**
  The spec marks it "optional polish for v0.4.0; full feature in v0.4.1"
  and the registry already exposes `list()` so the palette is purely UI
  layer. Deferred to v0.4.1.

- (S) **Pre-existing `console.warn` and bootstrap-error log noise** in
  `mqtt-store-bootstrap.spec.js` etc. is unchanged. Not in scope.
