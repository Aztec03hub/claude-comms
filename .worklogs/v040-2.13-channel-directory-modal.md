Added internal `$state` for tab + filter + admin-edit state, `$derived` for owned-channel gating, focus-management `$effect`, focus-trap keydown, Escape/overlay close paths, and styled the modal with the existing design tokens (`--bg-elevated`, `--border`, `--text-primary`, `--ember-500`, etc.).

The Browse tab mounts the (real) `ConversationBrowser` via the Step 2.14 contract props (`filterValue`, `sortKey="alphabetical"`, `embedded={true}`, `onChannelClick`). At integration time on main, Step 2.14's agent will have landed the refactor and the wire-up works end-to-end; until then the import resolves to the current ConversationBrowser, which we exercise in tests by stubbing `global.fetch` (the browser fetches `/api/conversations` on mount).

The Admin tab gates on `hasOwnedChannels = $derived(...)` — derived from `store.channelsById` filtered on `createdBy === store.userProfile.key`. When the user owns ≥ 1 channel the Admin tab button is visible (else hidden entirely from the tablist, matching the spec's "only visible if the user is creator of at least one channel"). Each owned-channel row exposes Edit topic / Archive / Delete affordances. Edit topic uses an inline `<input>` that commits via Enter or blur and cancels via Escape, calling `store.setTopic(id, newTopic)`. Archive calls `store.archiveChannel(id)` behind a `window.confirm` (v0.4.0 MVP — proper modal in v0.4.1). Delete calls `store.deleteChannel(id)` behind a `window.prompt` type-name confirm (also v0.4.0 MVP).

A11y: `role="dialog"` + `aria-modal="true"` + resolvable `aria-labelledby`; the tablist has `role="tablist"` + ARIA-managed tab indices; each tab is `role="tab"` with `aria-selected` + `aria-controls`; each panel is `role="tabpanel"` with `aria-labelledby`; focus trap cycles Tab and Shift+Tab between the first and last focusable descendants of the dialog; Escape closes via `onClose`; outside-click on the overlay closes; clicking inside the dialog content does not bubble; focus is restored to the previously-focused element on close (captured in a `$effect` on each open).

New test file `web/tests/channel-directory-modal.spec.js` covers all 12 required assertions (and adds 5 bonus tests for a total of 17): visibility, default tab, tab switch, admin gating both directions, initialTab="admin" hot start, ArrowRight tab nav, filter input behavior, initialFilter prefill, locked sort dropdown, owned-row admin buttons, close-button click, Escape, overlay click, content-click no-op, and Tab focus trap wrap.

### Internal state + derived ownership

```svelte
let activeTab = $state(untrack(() => (initialTab === 'admin' ? 'admin' : 'browse')));
let filterText = $state(untrack(() => initialFilter));

let ownedChannels = $derived(
  Object.values(store?.channelsById ?? {}).filter(
    (ch) => ch && ch.createdBy && ch.createdBy === store?.userProfile?.key,
  ),
);
let hasOwnedChannels = $derived(ownedChannels.length > 0);
```

`untrack(() => initialFilter)` is the Svelte 5 idiom for "use the prop's initial value to seed state without tracking the prop reactively" — fixes the `state_referenced_locally` autofixer warning.

### Focus-only $effect (state reset moved to close handler per Svelte 5 best practices)

```svelte
$effect(() => {
  if (!open) return;
  previouslyFocused = typeof document !== 'undefined' ? document.activeElement : null;
  queueMicrotask(() => {
    const activeNow = untrack(() => activeTab);
    if (activeNow === 'browse' && filterInputEl) {
      filterInputEl.focus();
    } else if (dialogEl) {
      const firstFocusable = dialogEl.querySelector('button:not([disabled]) ...');
      firstFocusable?.focus();
    }
  });
  return () => {
    if (previouslyFocused && ... ) previouslyFocused.focus();
  };
});

function close() {
  open = false;
  activeTab = initialTab === 'admin' ? 'admin' : 'browse';
  filterText = initialFilter;
  editingTopicId = null;
  editingTopicValue = '';
  onClose?.();
}
```
