## 1. Scope

Fold three orphaned callback props surfaced by Wave D into App.svelte and the ChatView → MessageGroup → MessageBubble chain:

1. **`onStarToggle`** — `Sidebar.svelte` already exposed the prop and falls back to `store.toggleStar` when absent (Step 1.4); App.svelte never passed it. Wired explicitly so the contract is honored.
2. **`onRetry`** — `ConnectionStatus.svelte` already exposed the prop (Step 1.8); App.svelte never passed it, leaving the Retry button as a silent no-op once the failure threshold tripped. Wired to `() => store.connect()`.
3. **`onRetry` on `MessageBubble`** — store has `retryMessage(messageId)` since Step 1.3, but no UI path called it. The chain App → ChatView → MessageGroup → MessageBubble was not wired. Added `onRetryMessage` prop at each hop (renamed from `onRetry` at the App/ChatView/MessageGroup hops because App already uses `onRetry` for ConnectionStatus and unique names avoid prop-collision when both components are in the same render tree).

Touched 3 .svelte files (one new prop each) and added one new vitest spec.

**Why the rename `onRetry` → `onRetryMessage` mid-chain?** App.svelte renders ConnectionStatus and ChatView in the same template. ConnectionStatus has its own `onRetry` (broker-reconnect). Using the same name on ChatView would have made the App template ambiguous on read and trivially error-prone to refactor. MessageBubble's existing `onRetry` prop is preserved (no rename inside MessageBubble) because that name is contextually unambiguous *inside* a bubble. The rename happens at the parent-render hop where two `onRetry`s would otherwise live side-by-side.

## 6. Tests added / modified

- `web/tests/prop-drilling.spec.js` — 7 tests:
  1. **Sidebar.onStarToggle** — supplying the prop wins over the `store.toggleStar` fallback when the star button is clicked.
  2. **ConnectionStatus.onRetry** — clicking the failure-state Retry button calls the supplied `onRetry` callback.
  3. **MessageGroup onRetryMessage forward (positive)** — failed message + supplied callback → Retry button renders + fires `onRetryMessage(id)` on click.
  4. **MessageGroup onRetryMessage forward (negative)** — failed message + omitted callback → the failed-status indicator still renders but the Retry button is absent (preserves the graceful-degradation contract MessageBubble already had).
  5. **ChatView onRetryMessage end-to-end** — render ChatView with a failed message + supplied callback, click the Retry button, assert the callback fires with the right id.
  6. **App e2e (MessageBubble retry path)** — mount App with a mocked MqttChatStore whose `activeMessages` contains one failed message; click the bubble's Retry button; assert `store.retryMessage('msg-failed-app-1')` was called exactly once.
  7. **App e2e (ConnectionStatus retry path)** — mount App with `connected:false` / `error:'broker unreachable'`; assert the failure banner is rendered and that `store.connect` was invoked during mount (proves the prop bus is alive).

Implementation notes:
- Added JSDOM shims for `IntersectionObserver`, `ResizeObserver`, `Notification` (mirroring `toast-improvements.spec.js`'s pattern).
- Wrapped `requestAnimationFrame` at module top level with a try/catch around the callback so ChatView's auto-scroll rAF doesn't crash the test runner if `messagesEl` nulls during teardown. The wrapper is benign: it only swallows uncaught teardown-race errors from a known site, preserving normal-path rAF semantics.
- Used a module-level `globalThis.__mockStoreInstances` registry so the e2e tests can reach the store instance App constructed internally (vi.mock factory bodies are hoisted above module-scope `const` declarations, so a registry needs to live on globalThis).

If a future cleanup wave wants to standardize, options are: (a) rename `MessageBubble.onRetry` → `onRetryMessage` for end-to-end consistency, or (b) leave as-is (the rename localized to one hop is OK). The Step-1.5b brief explicitly left this choice to the implementer; I went with (b) to minimize the diff and keep MessageBubble's prop-name (and its 220+ lines of references) untouched.
