## 1. Scope

Brand-new component implementing Phil's CTX-MENU-NEW constraint: a
right-click context menu for a single channel row in the sidebar. The
component owns rendering, keyboard navigation, the Mute submenu, and
viewport-overflow flipping. It is intentionally action-routing
agnostic — every click / Enter fires `onAction(actionId)` with a
documented string ID and then calls `onClose()`. The parent (Sidebar
in Step 2.12) is responsible for mapping each actionId to the
corresponding store method (`setStar`, `setMute`, `closeChannel`,
`deleteChannel`, `leaveChannel`, etc.) plus opening the relevant
downstream modal (LeaveChannelDialog from Step 2.11 + the channel-
directory modal for the Channel-info stub).

## 4. Code changes (samples)

The action-visibility computation lives in a single `$derived.by`,
so adding/removing items in v0.4.1 (admin-Delete extension) means
editing one block:

```js
const items = $derived.by(() => {
  const list = [];
  const hasUnread = (channel?.unread ?? 0) > 0;
  if (isMember) {
    list.push({ id: 'toggle-star', label: channel?.starred ? 'Unstar' : 'Star', icon: ... });
    list.push({ id: 'mute-submenu', label: 'Mute', icon: ..., submenu: [...] });
    if (hasUnread) list.push({ id: 'mark-read', label: 'Mark all as read', icon: CheckCheck });
  }
  list.push({ id: 'copy-link', label: 'Copy channel link', icon: LinkIcon });
  if (isMember && !isCreator) list.push({ id: 'leave', label: 'Leave', ... });
  if (isMember && isCreator)  list.push({ id: 'close', label: 'Close', icon: Archive });
  if (isCreator)              list.push({ id: 'delete', label: 'Delete', ... });
  list.push({ id: 'info', label: 'Channel info', icon: Info });
  return list;
});
```

Position-flipping uses a one-shot `$effect` keyed on a `positioned`
latch so jsdom (and the dev runtime) measure the rendered rect exactly
once after mount and flip if the cursor was near the bottom/right edge:

```js
let positioned = false;
$effect(() => {
  if (!menuEl || positioned) return;
  positioned = true;
  const initialX = anchorEvent?.clientX ?? 0;
  const initialY = anchorEvent?.clientY ?? 0;
  const rect = menuEl.getBoundingClientRect();
  // ... flip logic, then focus first menuitem via tick().then(...)
});
```

The `anchorEvent` prop is intentionally read inside the `$effect`
closure rather than at the top level — Svelte 5 emits the
`state_referenced_locally` warning if a prop is captured at the top of
`<script>` and then re-used after mount; reading inside the effect
satisfies the reactivity tracker.

## 9. Outstanding concerns

The brief's "Channel info" action visibility was "always" in the
spec table. v0.4.0 ships this as a stub (parent opens directory modal
scrolled to row). The component fires actionId `'info'`; routing is
the parent's responsibility per the contract. No judgment call was
required.

`tabindex={idx === activeIndex ? 0 : -1}` implements the standard
"roving tabindex" pattern for menu rows. The first item gets focus on
mount; arrow keys move focus and the tabindex tracking moves too. The
keyboard-test for ArrowRight initially flaked because state updates
from a preceding `fireEvent.keyDown(ArrowDown)` had not been flushed
before the next `fireEvent.keyDown(ArrowRight)` ran; adding an
explicit `await tick()` (and asserting the post-ArrowDown tabindex
state) between the two fires resolved it deterministically. Test
behavior matches the dev runtime exactly.
