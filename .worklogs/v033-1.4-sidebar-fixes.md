## 3. Design decisions / dead-ends

### Vite-define `__APP_VERSION__` rejected in favor of native JSON import

The task suggested using a Vite-define injected at build time. I tried that first, but discovered two friction points:

1. **vitest.config.js does not inherit `define`** from `vite.config.js` (vitest uses its own config when present). vitest.config.js was outside my writeable scope, so I couldn't add a mirror define there.
2. **The Svelte autofixer doesn't parse `with { type: 'json' }` import attributes** yet (returns "Unexpected token with"). This is the modern Stage 4 syntax but tooling lags.

**Resolution:** Use `import pkg from '../../package.json'` — bare specifier, no import attributes. Vite 6 resolves JSON imports natively via its built-in JSON plugin (no extra config). vitest inherits the same plugin chain. Autofixer parses cleanly. Belt-and-suspenders: the const expression falls back to empty string if `pkg?.version` is undefined.

```js
import pkg from '../../package.json';
const APP_VERSION = pkg?.version || '';
```

### Why I disambiguate G-4 with `starred` vs hollow visually instead of separate components

Lucide-svelte 1.0.1 does not ship a `StarFill` symbol — the "filled" variant is conveyed by setting `fill="currentColor"` on the `<Star>` SVG, which spreads via `{...$$props}` to the underlying `<Icon>`. Using one component with two render paths avoids a forked import surface and keeps the icon import list to the existing `Star` already in scope.

### CSS hover-visibility for the star toggle

The parent `.ch-actions` was already opacity-0 → opacity-1 on row-hover. For G-4 I needed the **filled** star to remain visible even off-hover, while leaving the mute button hover-only. Three rules accomplish this:

```css
.channel-item .ch-actions:has(.star-toggle.starred) { opacity: 1; }
.channel-item:not(:hover) .ch-actions .ch-action-btn:not(.star-toggle.starred) {
  opacity: 0;
  pointer-events: none;
}
.channel-item:hover .ch-actions .ch-action-btn {
  opacity: 1;
  pointer-events: auto;
}
```

The `:has()` selector is well-supported (Baseline 2023+). It promotes the wrapper to visible when any starred child exists, then per-child opacity strips visibility back from the non-starred siblings off-hover.

### G-25 three-state semantics

Mirrored from `ConnectionStatus.svelte`'s render branches:

| Store state                          | Label             | Class       | Dot color | Reference                |
| ------------------------------------ | ----------------- | ----------- | --------- | ------------------------ |
| `connected === true`                 | `"Online"`        | `online`    | green     | `.connection-dot` default |
| `!connected && connectionError`      | `"Offline"`       | `offline`   | red       | `.connection-dot.error-dot` |
| `!connected && !connectionError`     | `"Reconnecting…"` | `connecting`| amber     | `.connection-dot.connecting-dot` |

The store does not expose a single `connectionState` enum, so I derive the three-way label/class in the component via two `$derived` values. Color palette matches ConnectionStatus.svelte exactly (`#34d399`, `var(--ember-400)`, `#f87171`).

### `onStarToggle` prop with store fallback

The architecture doc says "the onStarToggle prop is already supported by the parent (check)." On inspection, App.svelte does **not** pass `onStarToggle` — only `onMuteChannel`. So I added the prop as optional (`onStarToggle = undefined`) with a runtime fallback: prefer the parent callback when provided, otherwise call `store.toggleStar(channelId)` directly. This keeps Sidebar functional today without an App.svelte edit (App.svelte is being modified by parallel step 1.5/1.7 — I avoided touching it).

**Before** (starred + unstarred sections — both identical):

```svelte
<div class="ch-actions">
  <button class="ch-action-btn" title={channel.muted ? 'Unmute' : 'Mute'} ...>
```

**After** (starred section — filled star):

```svelte

  <button
    class="ch-action-btn star-toggle starred"

    data-testid="channel-star-{channel.id}"
  >
    <Star size={10} fill="currentColor" />
  </button>
  <button class="ch-action-btn" title={channel.muted ? 'Unmute' : 'Mute'} ...>
    <VolumeX size={10} />
  </button>
</div>
```

**After** (unstarred section — hollow star, hover-only):

```svelte
<div class="ch-actions">
  <button
    class="ch-action-btn star-toggle"
    title="Star"
    aria-label="Star {channel.id}"
    aria-pressed="false"
    onclick={(e) => handleStarToggle(e, channel.id)}
    data-testid="channel-star-{channel.id}"
  >
    <Star size={10} />
  </button>
  <button class="ch-action-btn" title={channel.muted ? 'Unmute' : 'Mute'} ...>
    <VolumeX size={10} />
  </button>
</div>
```

**Handler updated to honor the optional prop:**

```js
// before:
function handleStarToggle(e, channelId) {
  e.stopPropagation();
  store.toggleStar(channelId);
}

// after:
function handleStarToggle(e, channelId) {
  e.stopPropagation();
  if (typeof onStarToggle === 'function') {
    onStarToggle(channelId);
  } else {
    store.toggleStar(channelId);
  }
}
```

with derived state:

```js
let connectionLabel = $derived(
  store.connected ? 'Online' : (store.connectionError ? 'Offline' : 'Reconnecting…')

let connectionState = $derived(
  store.connected ? 'online' : (store.connectionError ? 'offline' : 'connecting')
);
```
