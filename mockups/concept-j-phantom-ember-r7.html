Sure thing. The issue was in handleSessionRefresh() — when two requests hit simultaneously, both would pass the expiry check before either could update the token. Here's the fix with a mutex lock:
python 
import threading

# Lock to prevent race condition in session refresh
session_lock = threading.Lock()

def handle_session_refresh(session_id: str) -> dict:
    with session_lock:
        token = get_current_token(session_id)
        if token.is_expired():
            new_token = refresh_token(session_id)
            return {"token": new_token, "refreshed": True}
        return {"token": token, "refreshed": False}
